┌─────────────────────────────────────────────────────────┐

## 关键局限性

```
┌─────────────────────────────────────────────────────────────┐
│                    当前实现的局限性                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ❌ 子任务没有成功/失败判定                                   │
│     → 步数用完就切换，不管有没有做到                           │
│                                                             │
│  ❌ 历史记录不包含执行结果                                    │
│     → 规划器不知道上一步成没成功                               │
│     → 只能靠"看图猜"                                        │
│                                                             │
│  ❌ reward 信息没有传给规划器                                 │
│     → 规划器无法感知任务进展                                  │
│                                                             │
│  ✅ 主任务成功判定是可靠的                                    │
│     → 由 LIBERO 环境根据物体位置计算                          │
│                                                             │
│  ✅ 恢复规划机制有效                                         │
│     → 当规划器过早输出 done 时自动重试                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 数据流向

```
                    Qwen3-VL 规划器
                    ┌─────────────┐
                    │ 输入:       │
                    │ - 当前图像  │◄──── 摄像头观测
                    │ - 主任务    │◄──── "put both moka pots on stove"
                    │ - 执行历史  │◄──── "1. Place moka pot (15步)"
                    │             │      ⚠️ 没有成功/失败信息
                    │ 输出:       │
                    │ - 子任务指令│────► "Align right moka pot"
                    │ - 步数预算  │────► 10
                    │ - 或 done   │
                    └─────────────┘
                          │
                          ▼
                    π₀.₅ 低层策略
                    ┌─────────────┐
                    │ 输入:       │
                    │ - 当前图像  │◄──── 摄像头观测
                    │ - 子任务指令│◄──── "Align right moka pot"
                    │ - 机器人状态│◄──── 关节角度等
                    │             │
                    │ 输出:       │
                    │ - 连续动作  │────► [x, y, z, rx, ry, rz, grip]
                    │ (扩散模型)  │
                    └─────────────┘
                          │
                          ▼
                    LIBERO 仿真环境
                    ┌─────────────┐
                    │ 输入:       │
                    │ - 动作      │◄──── 关节控制指令
                    │             │
                    │ 输出:       │
                    │ - 新观测    │────► 新的摄像头图像
                    │ - reward    │────► 0.0 或 1.0
                    │ - is_success│────► True / False
                    │   (主任务)  │      ⚠️ 只判断主任务
                    └─────────────┘
```
